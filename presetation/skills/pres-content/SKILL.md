# Presentation Content Skill

## Overview

This skill writes the complete presentation script based on the structure document, using the Hristosenko selling methodology.

## Input Required

- `brainstorming.md` - Audience and product details
- `structure.md` - Presentation structure

## Core Methodology (MUST FOLLOW)

### The Content-Sales Paradox
**"В единицу нашего взаимодействия с человеком мы можем решить только одну задачу"**
(In one unit of interaction, we can only solve one task)

- You can either TEACH or SELL - not both simultaneously
- Content must serve sales, not replace it
- Replace "sales" with "persuasion" mentally

### The Ladder of Conviction (Лестница Убеждений)
Build conviction step by step:
1. Connect symptoms to the problem
2. Elevate problem to priority #1
3. Show your method is best
4. Show your product implements it best
5. Present specific offer options

### Content That Sells (NOT Educates)
For each content block:
- Talk about **WHY** it's important, not HOW to do it
- Show **SIGNIFICANCE** and **IMPACT**
- Give **INSIGHT** that there's an easier way
- **Make conclusions FOR them** (they'll make wrong ones otherwise)

## Script Writing Guidelines

### Opening Section

**Hook Formula:**
```
[Provocative statement or question that challenges their current belief]

"Что если я скажу вам, что всё что вы знали о [TOPIC] - неправильно?"
"What if I told you everything you knew about [TOPIC] was wrong?"
```

**Credibility Formula:**
- Brief (30-60 seconds max)
- Results-focused, not story-focused
- "За последние X лет я помог Y людям достичь Z"

### Problem Agitation Section

**Amplification Formula:**
```
1. Name the pain they feel
2. Show it's getting worse
3. Reveal hidden costs they don't see
4. Create urgency to solve NOW
```

**Key Phrases:**
- "Знакомо ли вам когда..." (Do you know the feeling when...)
- "А что будет если так продолжать..." (What happens if this continues...)
- "Большинство делают эти ошибки..." (Most people make these mistakes...)

### Content Blocks

**Each Block Must Have:**

1. **Significance Statement**
   - Why this matters
   - What changes when you get it right

2. **Contrast Example**
   - Wrong way vs right way
   - Before/after story

3. **Belief Shift**
   - Old belief: [What they currently think]
   - New belief: [What they should think]
   - Bridge: "На самом деле..." (Actually...)

4. **Product Bridge**
   - "Это один из элементов моей системы..."
   - Natural transition, not forced sales pitch

**NEVER Do:**
- Give step-by-step instructions
- Teach the "how" in detail
- Make them feel they can do it alone

### Transition Section

**The Bridge Formula:**
```
"Итак, вы узнали [3 things]..."
"So you've learned [3 things]..."

"Но чтобы получить [RESULT], нужно ещё [GAP]..."
"But to get [RESULT], you also need [GAP]..."

"Поэтому я хочу пригласить вас..."
"That's why I want to invite you..."
```

**Vaccination Technique (Прививка):**
Before presenting offer, acknowledge:
- "Сейчас будет продающая часть..."
- "Я знаю что вы не любите когда продают..."
- Then explain WHY you're selling (to help them get results)

### Offer Section

**Module Presentation:**
- Name modules with MARKETING names (results-focused)
- NOT methodological names (process-focused)
- Example: "Быстрый старт" vs "Введение"

**Value Stack Formula:**
```
Module 1: [NAME] - обычно стоит [X]
Module 2: [NAME] - обычно стоит [Y]
Bonus 1: [NAME] - стоит [Z]

ИТОГО: [TOTAL]

Но сегодня вы получаете всё это за [PRICE]
```

**Price Reveal:**
1. Anchor high (what it's worth)
2. Compare to alternatives
3. Reveal actual price
4. Show savings percentage

### CTA Section

**CTA Formula (Repeat 15+ times):**
```
"Нажмите кнопку ниже" + [Unique benefit each time]

CTA 1: "Нажмите кнопку и получите мгновенный доступ"
CTA 2: "Нажмите кнопку чтобы начать [transformation]"
CTA 3: "Нажмите кнопку пока действует цена"
...
```

**Urgency Elements:**
- Time deadline
- Price increase coming
- Bonus disappearing

**Scarcity Elements:**
- Limited spots
- Personal attention constraint
- Exclusive for event participants

## Output Format

Create: `project/{project-name}/script.md`

```markdown
# [PRODUCT NAME] Presentation Script

## OPENING (5-10 min)

### SLIDE 1: Title
[Speaker notes and exact script]

### SLIDE 2: Hook
[Script with timing]

...

## PROBLEM AGITATION (10-15 min)

### SLIDE X: Pain Point
[Script]

...

## CONTENT BLOCK 1 (8-12 min)

### SLIDE X: [Topic]
[Script with belief shift marked]

...

[Continue for all sections]
```

## Quality Checklist

- [ ] Every content block has a belief shift
- [ ] No "how-to" teaching (only "why")
- [ ] Transition uses vaccination technique
- [ ] Modules have marketing names
- [ ] 15+ CTA variations written
- [ ] Urgency AND scarcity elements included
- [ ] Price anchored before reveal
